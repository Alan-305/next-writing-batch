# 添削革命 — 教員向け利用マニュアル

最終更新: 2026-05-23  
対象: 教員・運用担当者（Google アカウントでログイン）

---

## このマニュアルについて

添削革命（next-writing-batch）は、生徒の英作文提出から AI 下書き添削、先生による確認・確定、生徒への返却までを支援する Web アプリです。

**Teacher-in-the-loop（先生が最終確認）** が前提です。AI の結果をそのまま返さず、必ず先生が内容を確認してから公開してください。

---

## 1. はじめに — ログインと基本画面

### 1-1. ログイン

1. 教員用 URL をブラウザ（Chrome 推奨）で開く  
2. **Google アカウント**でログインする  
3. 初回のみ **「教員としてテナントを作成」** を完了する  

### 1-2. テナント ID（organizationId）— 重要

画面右上に **テナント ID** が表示されます。

| ルール | 説明 |
|--------|------|
| **発行された ID をそのまま使う** | 提出・添削・生徒画面設定はすべてこの ID に紐づきます |
| **途中で ID を変えない** | ID を変えると、以前の提出や設定が「別テナント」になり一覧から見えなくなります |
| **生徒招待にも同じ ID を使う** | 「招待QRとチケット状況」画面のリンク・QR は、このテナントに紐づきます |

表示名（ニックネーム）とテナント ID は別物です。**データの所在はテナント ID で決まります。**

### 1-3. 運用ホーム（`/ops`）

日常作業は次のメニューから始めます。

| メニュー | 用途 |
|----------|------|
| **課題添削設定** | 課題内容・添削基準（JSON）の登録・更新 |
| **提出一覧** | 提出の確認、添削の「預ける」、詳細編集 |
| **納品 ZIP** | まとめてダウンロードした ZIP の確認 |
| **招待QRとチケット状況** | 生徒招待、チケット購入・残数確認 |
| **生徒画面設定** | 生徒が見る画面の色・学校名など |
| **問い合わせ** | 不具合・導入の相談 |

---

## 2. 生徒を招待する

1. **招待QRとチケット状況**（`/ops/tickets`）を開く  
2. **テナント ID** を控える（必要に応じてコピー）  
3. **招待リンク** または **QR コード** を生徒に共有する  
4. 生徒は Google ログイン後、初回のみプロフィール（学籍・氏名）を登録して提出画面へ進む  

生徒の提出があると、登録メールアドレスに **「新しい提出」** の通知メールが届きます（教員本人の試行提出は除く）。

---

## 3. 添削の基本 — 「今すぐ」より「預ける」

### 3-1. 結論（日常運用のおすすめ）

| ボタン | いつ使うか |
|--------|------------|
| **預ける**（おすすめ） | **通常はこちら。** 空き時間にサーバーが自動処理。メールで通知。 |
| **今すぐ** | 1 件だけ急ぎ確認したいときの例外。画面を開いたまま待つ必要がある。 |

**授業後や夜にまとめて「預ける」→ 翌朝メールで完了確認** という使い方が、時間をいちばん有効に使えます。

### 3-2. 「預ける」とは

- 添削ジョブを **クラウドのキューに預ける** 操作です  
- **ブラウザを閉じても PC をシャットダウンしても処理は続きます**  
- 処理は空き時間に順番に実行されます  

### 3-3. メール通知（登録した Google メールアドレス宛）

| タイミング | 内容 |
|------------|------|
| **預けた直後** | 「○ 件を受付しました」— 預けた件数の確認 |
| **処理中（長時間の場合）** | 約 **1 時間ごと** に途中経過（何件完了・残り何件） |
| **すべて完了** | 「預けていた添削がすべて完了しました」— 提出一覧で確認を促す |
| **生徒が新規提出** | 「新しい提出」— 課題 ID・学籍・氏名付き |

メール内の **提出一覧** リンクから、そのまま確認画面に進めます。

### 3-4. 提出一覧の状態表示

| 表示 | 意味 |
|------|------|
| **未添削** | まだ AI 添削していない |
| **待機中** | キューで順番待ち |
| **添削中** | AI が処理中 |
| **完了** | 添削成功。詳細で確認・編集へ |
| **要再実行** | 失敗または中断。**「預けてやり直し」** で再実行 |
| **閲覧済** | 生徒が結果ページを開いた |

添削中は **5 秒ごとに自動更新** されます。

### 3-5. 一括添削（提出一覧の上部パネル）

- **課題 ID を 1 つ選び**、その課題の未添削をまとめて預けられます  
- **1 回あたり最大 5 件** まで（安定運用のため）  
- 6 件以上ある場合は **5 件ずつ分けて** 「預ける」  
- 日常運用はパネルの **「預ける」** を使い、**「今すぐ（同期）」** は使わない  

行ごとの操作:

- 各行にも **「預ける」** ボタンがあり、**1 件だけ** 預けることもできます  

### 3-6. 「今すぐ」を使う場面（例外）

- 1 件だけ、**その場ですぐ** AI 出力を見たいデモ・確認  
- ネットワークやサーバー設定のトラブルシュート  

**注意:** 完了までブラウザを開いたまま待つ必要があり、件数が多いと非常に時間がかかります。

### 3-7. 要再実行になったとき

- AI 出力の形式不備などで失敗することがあります  
- **「預けてやり直し」** を押すと、多くの場合は再実行で成功します  
- 何度か失敗する場合は問い合わせフォームからご連絡ください

## 4. 添削後の流れ — 確認・確定・公開

添削が **完了** になったら、**詳細** ボタンから 1 件ずつ作業します。

### 4-1. 基本の 4 ステップ

```
AI 添削（預ける）
    ↓
先生が内容を確認・編集（下書き保存）
    ↓
確定（Day4 生成：音声・QR・PDF 用データ）
    ↓
生徒に公開する
```

| 操作 | 説明 |
|------|------|
| **下書き保存** | 得点・コメント・完成版英文を編集して保存 |
| **確定（Day4 生成）** | 文面をロックし、音声読み上げ・QR・PDF 用の成果物を生成 |
| **生徒に公開する** | 生徒が結果 URL で閲覧できる状態にする（Day4 成功後） |

**公開前に必ず内容を読み、必要なら修正してください。**

### 4-2. チケットについて

- 添削実行時点ではチケット残数を確認します  
- 教員プールのチケットは **Day4 確定（生徒向け成果物の確定）** のタイミングで **1 提出あたり 1 枚** 消費されます  
- 初回登録特典などで付与された枚数は **招待QRとチケット状況** で確認・追加購入できます

## 5. 紙で返却するとき — スマホ・パソコンの機能を使う

デジタル公開に加え、**印刷して紙で配る** 運用も可能です。専用の印刷ボタンはなく、**ブラウザ標準の印刷機能** を使います。

### 5-1. 先生側 — 詳細画面・成果物の確認

提出 **詳細** では Day4 生成後、次の成果物が確認できます。

- **音声**（模範解答の読み上げ）  
- **QR コード**（スマホで音声ページを開く用）  
- **PDF**（印刷向けレイアウト）  

### 5-2. 生徒向け結果ページを印刷する（先生が印刷して配布）

1. 公開済みの **生徒向け結果 URL** をブラウザで開く（詳細画面から確認可能）  
2. キーボードショートカットで **印刷** を開く  

| 環境 | 操作 |
|------|------|
| **Mac** | `⌘` + `P` |
| **Windows** | `Ctrl` + `P` |
| **スマホ・タブレット** | ブラウザのメニュー → **共有** または **印刷**（機種により「PDF に保存」も可） |

3. プリンタを選ぶか、**PDF に保存** してから印刷  
4. 印刷した用紙を生徒に配布  

結果ページには **印刷の案内** も表示されています。

### 5-3. 音声・QR を紙に載せる場合

- **QR コード** は詳細画面または PDF 右上に表示されます  
- 生徒は **自分のスマホのカメラ** で QR を読み取り、模範解答の **音声読み上げ** ページを開けます  
- 音声 QR の再生は **確定（Day4）から 30 日間** です（それ以降は聞けません）  
- 紙返却時は「英文＋QR」セットにすると、**耳から学ぶ** 運用（添削革命の指導方針）に合います  

### 5-4. 先生のスマホでの確認

- 提出一覧・メール通知は **スマホのブラウザ** からも利用できます  
- 外出先でメールを受け取り → リンクから提出一覧 → **預ける** だけ、という使い方も可能です  

---

## 6. 生徒画面設定

**生徒画面設定**（`/ops/student-appearance`）で、生徒が見る提出・結果画面のトーンを変更できます。

- 学校名・表示色など（テナント単位）  
- 設定は **テナント ID** に保存されます（ID を変えないこと）  

---

## 7. 課題添削設定

新しい課題を扱う前に **課題添削設定** で課題マスタ（課題 ID・ルーブリック・添削方針）を登録します。

- 課題 ID は提出一覧・一括添削で **同じ文字列** を使います  
- 画像・PDF から問題文を取り込む機能あり（取り込み後は必ず目視で修正）  

---

## 8. 1 日のおすすめワークフロー（例）

### 授業日

1. 生徒が提出（または写真・PDF で答案を送信）  
2. 先生に **新規提出メール** が届く  

### 授業後（5 分）

3. 提出一覧を開く  
4. 課題 ID を選び、未添削を **「預ける」**（最大 5 件ずつ）  
5. **受付メール** を確認してブラウザを閉じてよい  

### 翌朝または空き時間

6. **完了メール** を確認  
7. 提出一覧で **完了** の行を **詳細** から開く  
8. 内容を修正 → **確定** → **生徒に公開**  
9. 紙返却が必要なら **印刷（⌘P / Ctrl+P）** または PDF 保存

## 9. よくある質問

### Q. 預けたあと、一覧が「要再実行」になることがある

待機が長い場合や AI 出力の形式問題で表示が変わることがあります。**預けてやり直し** を試してください。データ自体が消えたわけではないことがほとんどです。

### Q. デプロイ（システム更新）で提出が消える？

**いいえ。** 提出データはクラウド上のデータベース（Firestore）に保存されます。更新後に見えない場合は、**テナント ID が以前と違う** 可能性を確認してください（右上表示）。

### Q. 「今すぐ」と「預ける」、どちらが品質が高い？

同じ AI エンジンです。**処理の速さと先生の待ち時間** が違います。通常は **預ける** で問題ありません。

### Q. 複数の課題 ID が混在している

一括パネルは **1 課題 ID ずつ** です。課題が違う場合は、課題ごとに預けるか、行ごとの **預ける** を使ってください。

### Q. メールが届かない

- 迷惑メールフォルダを確認  
- ログインに使っている **Google メールアドレス** が届先です  
- 届かない場合は問い合わせフォームへ  

### Q. 紙の QR を読むと「SignatureDoesNotMatch」や XML のエラーになる

- 昔の運用では QR に **Google Cloud の署名付き URL**（有効 **最大 7 日**）が入っていました。期限切れ後は「署名が合わない」というエラーになります。  
- **対処:** アプリを最新にしたうえ、該当提出を **もう一度「確定（Day4 生成）」** し、**新しい QR で再印刷** してください。

### Q. QR の音声はいつまで聞けますか

- **Day4 を確定してから 30 日間**（運用設定で変更可能）です。期限を過ぎると再生できません。  
- 長期保存用ではなく、**授業直後〜1 か月程度の学習用** を想定しています。それ以降も必要な場合は、期限内にダウンロードするか、運用に相談してください。
